28 ## Mem2Seq in pytorch
29 In this repository we implemented Mem2Seq and several baseline in pytorch (Version 0.3). To make the code more reusable we diveded each model in a separated files (obviusly there is a large code overlap). In the folder models you can find the following:
30 - ***Mem2Seq***: Memory to Sequence (Our model)
31 - ***Seq2Seq***: Vanilla seq2seq model with no attention (enc_vanilla)
32 - ***+Attn***: Luong attention attention model
33 - ***Ptr-Unk***: combination between Bahdanau attention and Pointer Networks ([Point to UNK words](http://www.aclweb.org/anthology/P16-1014))
34 
35 All of these file share the same structure, which is: a class that builds an encoder and a decoder, and provide training and validation methods (all inside the class).

40 ## Basic example
41 Mem2Seq can be considered as a general sequence to sequence model with the ability to address external memories. We prepared a very basic implementation (including data preprocessing and model) for a English to France translation task. Obviusly there is not much to copy from the input in this small corpus, so it is just to show how the model works in a general sequence to sequence task. Run:
42 ```console
43 ❱❱❱ python3 main_nmt.py
44 ```
45 This version uses a flat memory instead of triple as described in the paper.

47 ## Train a model for task-oriented dialog datasets
48 We created `main_train.py` to train models. You can see there is a notation, `globals()[args['decoder']]`, it is converting a string into a fuction. So to train a model you can run:
49 Mem2Seq bAbI t1-t6:
50 ```console
51 ❱❱❱ python3 main_train.py -lr=0.001 -layer=1 -hdd=128 -dr=0.2 -dec=Mem2Seq -bsz=8 -ds=babi -t=1
52 ❱❱❱ python3 main_train.py -lr=0.001 -layer=1 -hdd=128 -dr=0.2 -dec=VanillaSeqToSeq -bsz=8 -ds=babi -t=1
53 ❱❱❱ python3 main_train.py -lr=0.001 -layer=1 -hdd=128 -dr=0.2 -dec=LuongSeqToSeq -bsz=8 -ds=babi -t=1
54 ❱❱❱ python3 main_train.py -lr=0.001 -layer=1 -hdd=128 -dr=0.2 -dec=PTRUNK -bsz=8 -ds=babi -t=1
55 ```
56 or Mem2Seq In-Car
57 ```console
58 ❱❱❱ python3 main_train.py -lr=0.001 -layer=1 -hdd=128 -dr=0.2 -dec=Mem2Seq -bsz=8 -ds=kvr -t=
59 ❱❱❱ python3 main_train.py -lr=0.001 -layer=1 -hdd=128 -dr=0.2 -dec=VanillaSeqToSeq -bsz=8 -ds=kvr -t=
60 ❱❱❱ python3 main_train.py -lr=0.001 -layer=1 -hdd=128 -dr=0.2 -dec=LuongSeqToSeq -bsz=8 -ds=kvr -t=
61 ❱❱❱ python3 main_train.py -lr=0.001 -layer=1 -hdd=128 -dr=0.2 -dec=PTRUNK -bsz=8 -ds=kvr -t=
62 ```
63 
64 the option you can choose are:
65 - `-t` this is task dependent. 1-6 for bAbI and nothing for In-Car
66 - `-ds` choose which dataset to use (babi and kvr)
67 - `-dec` to choose the model. The option are: Mem2Seq, VanillaSeqToSeq, LuongSeqToSeq, PTRUNK
68 - `-hdd` hidden state size of the two rnn
69 - `-bsz` batch size
70 - `-lr` learning rate
71 - `-dr` dropout rate
72 - `-layer` number of stacked rnn layers, or number of hops for Mem2Seq
73 
74 
75 
76 While training, the model with the best validation is saved. If you want to reuse a model add `-path=path_name_model` to the function call. The model is evaluated by using per responce accuracy, WER, F1 and BLEU.

84 ## Notes
85 For hyper-parameter search of Mem2Seq, our suggestions are:
86 * Try to use a higher dropout rate (dr >= 0.2) and larger hidden size (hdd>=256) to get better performance when training with small hop (H<=3).
87 * While training Mem2Seq with larger hops (H>3), it may perform better with smaller hidden size (hdd<256) and higher dropout rate.
88 * Since there are some variances between runs, so it's better to run several times or run different seeds to get the best performance.
